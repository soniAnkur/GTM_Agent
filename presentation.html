↓
Deduplication check — comments already in MongoDB are skipped gracefully
↓
Gemini generates reply with full context: original post + incoming comment + 5 prior messages
6

↓

Idempotency check — comment looked up in MongoDB; skip if already processed

↓

6

Scheduling

Vercel Cron Jobs

Three cron jobs run the entire system autonomously. Secured with a CRON_SECRET header validated on every invocation.

*/5 * * * *
Post Scheduler
Every 5 minutes — checks for queued posts with scheduledAt <= now, validates rate limits, and publishes to Twitter/X, Facebook, and Instagram.
*/3 * * * *
Comment Monitor
Every 3 minutes — scans published posts from the last 7 days, fetches new comments via platform adapters, generates AI replies via Gemini, and posts them back.
0 */4 * * *
Content Generator
Every 4 hours — calls Gemini for platform-specific post copy, then kie.ai for a matching image. Stores generated content in queued_posts ready for the post scheduler.

Reliability

Error Handling Strategy

Every API call is wrapped in exponential backoff. Errors are categorized and handled deterministically — rate limits wait, auth errors skip, network errors retry.

429 Rate Limit
Exponential backoff — base 2s, factor 2x, max 3 attempts with jitter. If max retries exceeded, mark post as failed and alert.
401 / 403 Auth Error
Log error and skip the platform entirely for this execution cycle. Does not block other platforms.
Network Error
Retry up to 3 times with exponential backoff. If all retries exhausted, mark the post as failed in MongoDB.
400 Validation Error
Log immediately and mark post as failed. No retries — the request is structurally wrong.
// src/lib/utils/retry.ts — Exponential backoff with jitter async function withRetry<T>( fn: () => Promise<T>, opts = { maxAttempts: 3, baseDelayMs: 2000, factor: 2 } ): Promise<T> { for (let attempt = 1; attempt <= opts.maxAttempts; attempt++) { try { return await fn(); } catch (err) { if (attempt === opts.maxAttempts) throw err; const delay = opts.baseDelayMs * (opts.factor ** (attempt - 1)) + Math.random() * 1000; // jitter await sleep(delay); } } }
Security

Security Architecture

Defense-in-depth across webhook verification, cron authentication, secrets management, and database access controls.

🔐
Webhook Verification
HMAC-SHA256 signature check on every Meta webhook event using X-Hub-Signature-256 header.
🔑
Cron Authentication
All /api/cron/* endpoints validate the Vercel CRON_SECRET header before executing.
🌿
Environment Secrets
All API keys stored in .env.local (gitignored). Zero credentials in source code.
🗄️
MongoDB Security
Authenticated connection string with optional IP whitelist. Singleton client pattern prevents connection exhaustion.
